# Archived: OttoBot Agent-Bridge Planning

This folder contains planning documents for a proposed 'agent-bridge' microservice
that would enable an LLM (OttoBot) to play tic-tac-toe on OttoChain.

**Status**: Planning only — implementation never started
**Archived**: 2026-02-03
**Reason**: Superseded by identity-service work; keeping as reference material

## Contents
- OTTO_PERSONA.md — LLM system prompt (useful reference for agent integration)
- ARCHITECTURE.md — Agent-bridge architecture design
- TOOL_CATALOG.md — Tool definitions for LLM function calling
- API_REFERENCE.md — Proposed HTTP API
- SETUP_GUIDE.md — Installation guide (references unbuilt code)
- PROJECT_STATUS.md — Phase tracking

## Relevance
The patterns here (tool calling, state machine interaction, persona design) 
are applicable to the agent identity service being developed.
