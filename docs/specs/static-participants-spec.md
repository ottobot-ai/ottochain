# Static Test Participants Spec

## Goal
Add a `fromResources` factory to `ParticipantRegistry` that loads 26 pre-generated p12 keystores from test resources, providing deterministic addresses for tests that need token balances (via genesis.csv).

Keep the existing `create[F]()` method for tests that don't care about tokens/addresses.

## Files Already Created
- `modules/shared-test/src/main/resources/participants/participant_1.p12` through `participant_26.p12` — generated by tessellation's cl-keytool
- `modules/shared-test/src/main/resources/participants/participants.csv` — name,index,alias,password,address mapping
- `modules/shared-test/src/main/resources/participants/genesis.csv` — address,balance for test genesis (1B tokens each)
- All keystores use password `testpassword`, alias `participant_N`

## Changes Required

### 1. `Participant.scala` — Add `fromResources` to `ParticipantRegistry`

```scala
object ParticipantRegistry {
  // KEEP existing create method unchanged
  def create[F[_]: Async](participants: Set[Participant])(implicit s: SecurityProvider[F]): F[ParticipantRegistry[F]]

  // NEW: Load from static p12 keystores in resources
  def fromResources[F[_]: Async]()(implicit s: SecurityProvider[F]): F[ParticipantRegistry[F]]
}
```

The `fromResources` method should:
1. Load all 26 p12 files from classpath resources at `participants/participant_N.p12`
2. Map them to the existing `Participant` case objects (Alice=1, Bob=2, ..., Zoe=26)
3. Extract `KeyPair` from each p12 using tessellation's `KeyStoreUtils.readKeyPairFromStore`
4. Build `ParticipantData` with address + proof function (same as `create` does)
5. Return a `ParticipantRegistry` with all 26 participants

### 2. Loading p12 from classpath

Use tessellation's existing `KeyStoreUtils`:
```scala
import io.constellationnetwork.keytool.KeyStoreUtils

// Load from classpath resource
val resourcePath = s"participants/participant_$index.p12"
val inputStream = getClass.getClassLoader.getResourceAsStream(resourcePath)
// Write to temp file (KeyStoreUtils.readKeyPairFromStore takes a file path)
// OR use KeyStoreUtils directly if it supports InputStream
```

Check `KeyStoreUtils` for the exact loading API. It may need a file path, in which case write the resource to a temp file first. The password and alias are both known constants.

### 4. Test
Add a test in `shared-test` (or `shared-data` test scope) that:
- Creates a registry via `fromResources()`
- Verifies Alice's address matches `DAG6HdXmFyEwgKKdaEyAjU6SJPxGNAjUSbHgiRct`

## Important Notes
- Do NOT modify the existing `create` method — tests using random keys should keep working
- The p12 files are already in the resources directory — don't regenerate them
- Use `testpassword` as both store password and key password
- Alias format is `participant_N` (e.g., `participant_1`, `participant_2`)
- This is on `scasplte2/ottochain` — branch from `origin/main`
- Use `source ~/.sdkman/bin/sdkman-init.sh` before any sbt commands
- Run `sbt sharedTest/test` to verify tests pass
